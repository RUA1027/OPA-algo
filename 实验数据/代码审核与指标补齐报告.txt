二、硬件实验代码审核结论

1. 从图像提取相位方向和波长方向 FWHM：
原代码已经有 func_extract_FWHM.m，并且 exp_update_realtime_display.m 会在最佳图像刷新时调用它。但是原实现使用简单阈值扫描，遇到半高位置落在像素之间、边界未跌破半高、噪声或平台峰时容易得到不稳定宽度。

本轮已修复：
func_extract_FWHM.m 改为基于峰值所在行/列的一维剖面，使用半高位置线性插值计算 FWHM。若图像为空、全非有限值或峰值无效，返回 NaN，而不是给出错误宽度。

2. 从图像定位峰值位置：
原代码已经有 func_locate_peak.m，但采用双层循环并在多个最大值时返回最后一个最大点。

本轮已修复：
func_locate_peak.m 改为直接使用 max(image(:)) 和 ind2sub 定位峰值，记录 x_pixel、y_pixel、x_deg、y_deg、intensity。func_locate_wl_position.m 复用 func_locate_peak，避免两套峰值定位逻辑不一致。

3. 峰值位置偏差 / 目标点偏差 + phase 方向 FWHM + wl 方向 FWHM：
原代码只把 bestPhaseFwhmDeg、bestWlFwhmDeg、bestBeamPositionDeg 写入部分算法结果，bestBeamPositionDeg 只代表波长方向位置，不是二维峰值，也没有目标点偏差。

本轮已补齐：
exp_update_realtime_display.m 在最佳图像刷新时记录二维峰值位置、二维角度位置、目标点像素偏差、目标点角度偏差和 RSS 偏差。
exp_init_realtime_display.m 增加 targetXPixel、targetYPixel、bestPeak、bestTargetDeviation 等字段。
exp_run_calibration.m 将 cfg.measure.ccdCenterCol 和 cfg.measure.ccdTargetRow 传入实时显示配置。
exp_run_wavelength_calibration.m 在每个目标点运行时设置 cfgPoint.measure.ccdTargetRow = target.y_pixel，因此 target x/y 都进入评价链路。
新增 exp_attach_display_metrics.m，统一把实时显示得到的图像指标写回 result，避免 5PPS、mREV、mREV-GSS、PFPD 四个实验 runner 字段不一致。

4. 自动化统计脚本：
原数据处理目录已有 batch_analyze_experiments.m 和 compute_spot_metrics.m，可批量扫描 best_image.tif 和 result.mat，输出 summary.csv。但原脚本没有输出按算法分组均值表，失败行默认值用 0，容易和真实值混淆；FWHM 算法也继承了旧的简单阈值逻辑。

本轮已补齐：
compute_spot_metrics.m 使用与硬件侧一致的插值 FWHM 和峰值定位逻辑。
batch_analyze_experiments.m 将缺失值初始化为 NaN，输出新增字段 peak_x_deg、peak_y_deg、deviation_x_deg、deviation_y_deg、deviation_rss_deg。
batch_analyze_experiments.m 继续生成 数据处理\summary.csv，并新增 数据处理\summary_stats.csv，记录每种算法的平均 evalCount、平均耗时、平均最终 FOM、平均最佳强度、平均目标偏差、平均 phase-FWHM、平均 wl-FWHM。

四、结果测评环节指标落地情况

1. 已经在结果中存在且代码明确实现的指标：
收敛速度/迭代次数：evalCount、roundEvalCount、roundTargetIntensityTrue。
总耗时：硬件侧 calibrationElapsedSec / manifest elapsed_sec；仿真侧 runtimeSec。
最终 FOM：硬件侧 finalIntensityMeasured / final_intensity；仿真侧 targetIntensity。
最佳强度：硬件侧 best_image 的峰值 / best_intensity；仿真侧 mainPeak。
FWHM / 光束发散角：硬件侧 phase_FWHM、wl_FWHM；仿真侧 fwhmDeg / beamwidth3dBDeg / beamDivergenceDeg。
几何对准精度：硬件侧 peak_x/y 与 target_x/y 的 deviation；仿真侧 pointingErrorDeg / pointingErrorAbsDeg。
副瓣抑制或主副瓣对比：仿真侧 smsrDb、slsrDb、mainSideContrastDb。硬件 CCD 近场图像不是远场角谱，当前不能直接从现有硬件图像严谨计算 SLSR/SMSR。

2. 需要测量过程中同步获得的指标：
evalCount、roundEvalCount、calibrationElapsedSec、runtimeSec、测量 FOM 曲线、最终测量值、最佳控制电压、最佳 CCD 图像。这些指标必须在优化运行时记录，否则事后无法完全恢复。

3. 可以测量结束后离线分析得到的指标：
从 best_image.tif 或 best_image 矩阵得到的二维峰值、phase/wl FWHM、目标偏差、图案拼接质量统计。只要图像和目标坐标保存完整，这些可以离线批量重算。

4. 硬件侧暂不应直接声称已实现的指标：
SLSR/SMSR 或主副瓣对比度。当前硬件实验保存的是 CCD 图像和积分强度，缺少严格远场角谱或足够定义主副瓣区域的光学标定链路。若论文需要硬件 SLSR/SMSR，建议增加远场角度映射标定或在采集端保存可用于旁瓣分析的完整远场图像，并明确主瓣/副瓣分割方法。
