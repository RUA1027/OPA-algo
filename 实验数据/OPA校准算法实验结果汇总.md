# OPA校准算法实验结果报告

## 1. 实验目标与数据范围

本报告基于当前目录下全部实验输出数据，对 OPA 校准算法结果进行系统化汇总与分析，覆盖两部分任务：

1. 单点校准对比：5PPS 与 mREV 在同一目标点上的性能对比。
2. 多点打点任务分析：笑脸任务（smile）与 ZJU 字母任务（zju），两者均采用 5PPS。

数据源统计如下：

- `run_manifest.csv` 共 24 份。
- 点结果（manifest 行）共 56 条。
- 单点对比：2 条（5PPS 1 条，mREV 1 条）。
- 笑脸任务：14 次运行，26 条点结果（24 个唯一点位，C8/C12 各重复一次）。
- ZJU 任务：8 次运行，28 条点结果（28 个唯一点位）。
- 所有 manifest 记录状态均为 `completed`。

## 2. 指标定义与统计口径

### 2.1 原始字段来源

- 运行清单：`*/run_manifest.csv`
- 点级详情：`point_*_result.mat`
- 波长级汇总：`wavelength_run_result.mat`

### 2.2 关键指标

1. 优化时间：`calibrationElapsedSec`（若缺失则使用 manifest 的 `elapsed_sec`）。
2. 评估次数：`evalCount`。
3. 迭代轮数：`roundCount`。
4. 强度：`initialIntensityMeasured` 与 `finalIntensityMeasured`。
5. 强度增益比：$G=I_{final}/I_{initial}$。
6. 光斑宽度与面积（由 `best_image` 统一计算）：
   - 背景估计：图像 5% 分位值。
   - 净信号：$I_{net}=\max(I-I_{bg},0)$。
   - 横/纵向宽度：用二阶矩估计 $\sigma_x,\sigma_y$，再换算 $FWHM=2\sqrt{2\ln2}\,\sigma$。
   - 半高面积：$I_{net}\ge0.5\cdot I_{net,max}$ 的像素数。

说明：上述口径对全部点结果一致，便于跨任务横向比较。

## 3. 单点校准对比（5PPS vs mREV）

测试对象：`S2 (x=160, y=175)`，输入波长 1561.781 nm（matched 同为 1561.781 nm）。

### 3.1 核心指标对比

| 指标                      |        5PPS |        mREV | 差异说明                         |
| ------------------------- | ----------: | ----------: | -------------------------------- |
| 优化时间                  |   302.704 s |  6070.122 s | mREV/5PPS=20.053，差异=+1905.30% |
| 评估次数(evalCount)       |     1793 次 |    76159 次 | mREV/5PPS=42.476，差异=+4147.57% |
| 迭代轮数(roundCount)      |        4 轮 |        4 轮 | 轮数相同                         |
| 初始强度                  | 386072 a.u. | 377274 a.u. | mREV 略低                        |
| 最终强度                  | 630654 a.u. | 640031 a.u. | mREV +1.49%                      |
| 强度增益比(final/initial) |      1.634x |      1.696x | mREV +3.85%                      |
| 光斑半高面积              |       22 px |       25 px | mREV +13.64%                     |
| FWHM_x                    |  140.174 px |  139.223 px | mREV -0.68%                      |
| FWHM_y                    |   73.201 px |   75.139 px | mREV +2.65%                      |
| 峰值净强度                |  11059 a.u. |  10941 a.u. | 5PPS 略高                        |

### 3.2 轮次行为对比

| 算法 | 每轮评估次数               | 每轮强度结果                     | 观察                             |
| ---- | -------------------------- | -------------------------------- | -------------------------------- |
| 5PPS | [448, 448, 448, 448]       | [509419, 579833, 625827, 630763] | 评估预算均匀，收敛节奏稳定       |
| mREV | [1856, 7231, 18623, 48448] | [562574, 625509, 637325, 639465] | 后两轮评估急剧上升，长尾搜索明显 |

### 3.3 结论

1. mREV 在最终强度上仅有小幅优势（约 +1.49%），但代价是显著增加计算量与时长（时间约 20 倍、评估次数约 42 倍）。
2. 若以工程效率和在线可用性为目标，5PPS 性价比更高。
3. 若追求极限强度且离线时间充足，可考虑 mREV 作为二阶段精修，但建议增加早停策略抑制长尾计算。

补充效率指标：

- 单位时间强度提升：5PPS 约 808 a.u./s，mREV 约 43.3 a.u./s，5PPS 明显更优。

## 7. 总结与优化建议

### 7.1 主要结论

1. 单点对比中，5PPS 与 mREV 最终强度接近，但 5PPS 在计算效率上优势极大，更适合工程化部署。
2. 两个打点任务在优化时间上均稳定在约 5 分钟/点量级。
3. 笑脸任务在强度水平与均匀性上均优于 ZJU 任务；ZJU 任务完整性高，但亮度离散较大。

### 7.2 建议

1. 以 5PPS 作为默认主算法；仅在离线高精度场景启用 mREV 精修，并增加早停阈值。
2. 对 ZJU 任务引入按字母或按行分组的曝光/驱动补偿，降低点间强度波动。
3. 对重复点位（如 smile 的 C8/C12）建立自动去重和版本标注，避免重复运行干扰统计。
4. 在图像拼接后增加统一的热点抑制与连通域后处理，提高字符边界清晰度。
